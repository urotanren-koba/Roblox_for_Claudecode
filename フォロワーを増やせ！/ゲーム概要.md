| 項目 | 内容 |
|------|------|
| プラットフォーム | Roblox |
| ジャンル | ペット育成 × フォロワー獲得シミュレーター |
| ターゲット | 10代、インフルエンサーに憧れる層 |
| タイトル候補 | フォロワーを増やせ！ / Follower Simulator / インフルエンサーシミュレーター |

---

## MVP（最小構成）

最初にリリースする最小限の機能：

```
ガチャで卵を引く → ペットが出る → レア度に応じてフォロワー増加
```

これだけでゲームとして成立させ、反応を見てから機能追加する。

## コアシステム

### 1. ガチャシステム

| 種類 | 内容 |
|------|------|
| 無料ガチャ | 1日1回、または広告視聴後 |
| 通常ガチャ | 石を消費して卵を入手 |
| レアガチャ | 多めの石を消費、星4以上確定など |

### 2. ペットシステム

| 項目 | 仕様 |
|------|------|
| 入手方法 | ガチャで卵を引く |
| レア度 | 星1〜星5 |
| 表示 | プレイヤーの後ろについてくる |
| データ管理 | ReplicatedStorageに保存、表示時のみClone() |
| モデル | ツールボックスから入手（Low Poly推奨、スクリプト削除必須） |

### 3. フォロワーシステム

レア度に応じたフォロワー獲得：

| レア度 | 獲得フォロワー数（例） |
|--------|----------------------|
| 星1 | 5人 |
| 星2 | 20人 |
| 星3 | 100人 |
| 星4 | 500人 |
| 星5 | 1,000人 |

### 4. 石の入手方法

| 方法 | 獲得量 |
|------|--------|
| デイリーログイン | 少量 |
| ミッション達成 | 中量 |
| イベントクリア | 多め |
| 課金（Robux） | 大量 |

## エリア・部屋システム

フォロワー数に応じて入れるエリアが変わる（サブミッション的位置づけ）：

| フォロワー数 | エリア | 雰囲気 |
|-------------|--------|--------|
| 0〜1,000 | 初心者スタジオ | 狭い、機材少ない |
| 1,000〜10,000 | クリエイター部屋 | そこそこ豪華 |
| 10,000〜100,000 | 人気配信者ラウンジ | 広い、装飾あり |
| 100,000〜 | VIPセレブルーム | 超豪華、限定アイテム |

### VIPルームの特典

- レア素材が落ちる（育成効率アップ）
- 限定ペットが手に入る
- 育成速度ブースト
- コラボ相手（NPC）がいる

|-------------|-----|--------|
| 1,000 | ブロンズ盾 | 銅、シンプル |
| 10,000 | シルバー盾 | 銀、少し光る |
| 100,000 | ゴールド盾 | 金、キラキラ |

### 特殊盾（今後追加候補）

- 連続ログイン盾（7日/30日）

## 技術メモ

### サーバー構造

- 1サーバー最大50〜100人
- 共有エリア（ロビー、VIPルーム）：全員同じ場所
- ペットは物理的な「部屋」不要、データのみで管理

### ツールボックス使用時の注意

1. スクリプトは全削除（セキュリティ）
2. パーツ数確認（30以下が理想）
3. 重い場合はMeshに変換
4. 「Low Poly」で検索すると軽いモデルが見つかる

### データ保存

- DataStoreServiceでプレイヤーごとのデータ保存
- 保存項目：所持ペット、フォロワー数、石の数、盾獲得状況

## 成功の鍵（Roblox人気ゲームの特徴）

1. **最初の30秒で何すればいいかわかる** - チュートリアル不要
2. **数字が増える快感** - フォロワーが増える＝進んでる感
3. **ループが短い** - 1プレイ3〜5分で完結
4. **見た目でわかる成長** - ペット、盾で「すごい」と思われる
5. **ソーシャル要素** - VIPルームで見せびらかせる
